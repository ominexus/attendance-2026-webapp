# 2026 고등부 출석부 웹서비스 프로젝트 구상

## 1. 기존 Google Sheets 출석부 분석

Google Drive에 저장된 `2026 고등부 출석부.xlsx` 파일을 분석한 결과, 해당 문서는 교사 및 학생의 기본 정보를 관리하고 매주 일요일의 출석 여부를 기록하며 이를 통계화하는 구조로 이루어져 있습니다.

### 1.1. 시트 구성 및 역할

전체 워크북은 명단 관리, 반별 출석 기록, 통계 산출, 그리고 입력 인터페이스 역할을 하는 여러 시트로 유기적으로 연결되어 있습니다.

| 시트 분류 | 시트명 | 주요 역할 및 특징 |
| :--- | :--- | :--- |
| **명단 관리** | 교사 명단, 학생 명단 | 교사의 직분 및 담당 반 정보, 학생의 기본 인적 사항(학년, 반, 성별, 연락처 등)을 중앙에서 관리합니다. |
| **출석 기록** | 1-1 ~ 3-2 (반별 시트), 전체학생 | 각 반별 학생 목록과 주차별 출석 여부(True/False)를 기록합니다. `VLOOKUP` 및 `FILTER` 함수를 통해 명단 시트에서 데이터를 동적으로 불러오며, `COUNTIF`로 총 출석 횟수를 계산합니다. |
| **통계 산출** | 출결 통계 | 각 반별 주차별 출석 인원 합계 및 전체 합계를 집계합니다. `HLOOKUP`과 `SUM` 함수를 사용하여 반별 시트의 데이터를 참조합니다. |
| **입력 인터페이스** | 출석입력 | 특정 날짜, 학년, 반을 선택하여 해당 반 학생들의 출석을 체크박스 형태로 입력하는 화면입니다. |

### 1.2. 핵심 워크플로우

기존 시스템은 중앙 집중화된 명단 데이터를 바탕으로 작동합니다. 사용자가 '출석입력' 시트에서 날짜와 반을 지정한 뒤 체크박스를 통해 출석 여부를 입력하면, 이 데이터가 각 반별 시트와 '전체학생' 시트에 동기화됩니다. 최종적으로 '출결 통계' 시트에서 주차별, 반별 출석 인원이 자동으로 집계되는 흐름을 가집니다.

## 2. 웹서비스 프로젝트 구상 (GitHub + Supabase 기반)

기존 스프레드시트 방식은 동시 편집 시 충돌이 발생할 수 있고, 모바일 환경에서의 접근성이 떨어지며, 데이터 무결성을 보장하기 어렵다는 단점이 있습니다. 이를 해결하기 위해 사용자의 기술 스택(Android, Supabase, React 등)에 맞춘 웹서비스 아키텍처를 제안합니다.

### 2.1. 시스템 아키텍처

본 프로젝트는 프론트엔드와 백엔드를 분리한 서버리스 아키텍처를 채택합니다. 프론트엔드는 React와 Vite, TailwindCSS를 사용하여 구축하고 GitHub Pages를 통해 호스팅합니다. 백엔드 및 데이터베이스는 Supabase를 활용하여 PostgreSQL 기반의 관계형 데이터베이스와 REST API, 인증 시스템을 통합적으로 제공받습니다. 소스 코드의 버전 관리는 GitHub를 사용합니다.

### 2.2. 데이터베이스 스키마 설계 (Supabase PostgreSQL)

데이터의 정규화와 무결성을 위해 다음과 같이 핵심 테이블을 설계합니다.

| 테이블명 | 설명 | 주요 컬럼 |
| :--- | :--- | :--- |
| `teachers` | 교사 및 직분 정보 | `id` (PK), `role` (직분/반), `name` (이름) |
| `students` | 학생 기본 정보 | `id` (PK), `grade` (학년), `class_num` (반), `name` (이름), `gender` (성별), `phone` (전화번호), `birth_date` (생일), `school` (학교), `guide` (인도자) |
| `attendance` | 출석 기록 | `id` (PK), `student_id` (FK), `attendance_date` (출석일), `status` (출석여부 boolean) |

### 2.3. 핵심 기능 구현 방안

출석 입력 인터페이스는 모바일 환경에 최적화하여 개발합니다. 날짜 선택(기본값: 가장 최근 일요일)과 학년/반 필터링 기능을 제공하며, 학생 목록을 리스트 형태로 표시하여 토글 버튼(Switch)으로 출석 상태를 직관적으로 변경할 수 있게 합니다. 변경 사항은 Supabase REST API를 통해 즉시 데이터베이스에 반영되며, Optimistic UI를 적용하여 사용자 체감 반응 속도를 높입니다.

대시보드 및 통계 기능은 기존의 '출결 통계' 시트를 대체합니다. Supabase의 SQL View 또는 RPC(Remote Procedure Call)를 활용하여 주차별/반별 출석 통계 데이터를 서버 측에서 집계하고, 프론트엔드에서는 Recharts 또는 Chart.js를 사용하여 시각적인 그래프로 제공합니다.

명단 관리 기능은 학생 및 교사 정보의 생성, 조회, 수정, 삭제(CRUD)를 지원합니다. 초기 데이터 마이그레이션의 편의성을 위해 엑셀 또는 CSV 파일 업로드를 통한 일괄 등록 기능을 포함합니다.

### 2.4. 보안 및 권한 관리

Supabase의 Row Level Security (RLS) 기능을 적용하여 데이터 보안을 강화합니다. 인증된 사용자(교사 및 관리자)만이 데이터를 조회하고 수정할 수 있도록 제한하며, 필요에 따라 교사는 본인이 담당하는 반의 데이터만 수정할 수 있도록 세밀한 권한 제어 정책을 설정합니다.

### 2.5. 개발 단계 (Milestones)

프로젝트는 총 4단계로 나누어 진행합니다. 첫 번째 단계에서는 Supabase 프로젝트를 생성하고 테이블 및 RLS를 설정한 뒤, 기존 엑셀 데이터를 파싱하여 데이터베이스로 마이그레이션합니다. 두 번째 단계에서는 Vite와 React를 이용해 프론트엔드 뼈대를 구축하고 Supabase Client 연동 및 로그인 기능을 구현합니다. 세 번째 단계에서는 출석 체크 페이지와 통계 대시보드 등 핵심 기능을 완성합니다. 마지막 네 번째 단계에서는 GitHub Actions를 활용해 GitHub Pages 자동 배포 파이프라인을 구축하고, 모바일 환경에서의 UI/UX 테스트 및 최적화를 수행합니다.
